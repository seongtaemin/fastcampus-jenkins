Jenkins 및 Gitea 파이프라인 구성 가이드

본 문서는 Jenkinsfile-3tier와 Jenkinsfile-argocd 파이프라인을 온전히 실행하기 위해 필요한 플러그인, 시스템 설정, 그리고 Credentials 등록 절차를 안내합니다.

1단계. 필수 플러그인 설치

Jenkins 관리(Manage Jenkins) > 플러그인 관리(Plugins) > Available plugins 탭에서 다음 플러그인들을 검색하여 설치합니다.

  • SonarQube Scanner (정적 코드 분석용)
  • Credentials Binding (Credentials를 스크립트 변수로 바인딩하기 위해 필수)
  • Pipeline (기본 설치)
  • (선택) Gitea Plugin (Gitea Webhook 연동을 통한 자동 빌드 구성 시 필요)

2단계. Jenkins 시스템 및 글로벌 도구 설정

2.1. SonarQube Scanner 도구 등록

  1. Jenkins 관리 > Tools (글로벌 도구 설정)로 이동
  2. SonarQube Scanner 섹션에서 Add SonarQube Scanner 클릭
  3. Name: sonar-scanner 입력 (Jenkinsfile에 하드코딩된 이름과 매칭 필수)
  4. Install automatically: 체크
  5. 저장(Save)

2.2. SonarQube Server 등록

  1. Jenkins 관리 > System (시스템 설정)로 이동
  2. SonarQube servers 섹션에서 Environment variables 체크
  3. Add SonarQube 클릭
  4. Name: sonarqube-server 입력
  5. Server URL: SonarQube의 URL 입력 (예: http://sonarqube:9000)
  6. 저장(Save)
*참고: Jenkins 에이전트 환경에 기본적으로 docker, kubectl, helm, curl가 설치되어 있어야 합니다.

3단계. Credentials (자격 증명) 등록

Jenkins 관리 > Manage Credentials > (global) > Add Credentials에서 다음 항목들을 등록합니다.

ID (정확히 입력) Kind (타입) 입력 내용 (Description)
harbor-credentials Username with password - Username: Harbor 로그인 ID (예: admin)
- Password: Harbor 패스워드
github-token Username with password (이름은 github이지만 실제 Gitea 정보를 입력)
- Username: Gitea 사용자 ID
- Password: Gitea 비밀번호 또는 Access Token
argocd-credentials Username with password - Username: admin (ArgoCD ID)
- Password: ArgoCD 초기 비밀번호
managed-cluster-kubeconfig
(임의 지정 가능)		 Secret file - File: 클러스터에 접근할 수 있는 kubeconfig 파일 업로드
* Jenkinsfile-argocd 빌드(Build with Parameters) 시 선택할 항목입니다.

4단계. Gitea 연동을 위한 Jenkinsfile-3tier 코드 수정

Jenkinsfile-3tier 파일 내에 하드코딩된 GitHub 저장소 URL을 사용하는 환경 구축용 Gitea 주소로 변경해야 합니다.

1. 환경 변수 (Environment) 영역 수정

// 변경 전 (약 10번째 줄)
GITOPS_REPO_URL = 'https://github.com/seongtaemin/3tier-app.git'

// 변경 후 (실제 Gitea 주소와 사용자로 변경)
GITOPS_REPO_URL = 'http://<Gitea서버IP:포트>/<사용자명>/3tier-app.git'

2. Argo Git Push 스테이지 하단(git push 명령어) 영역 수정

# 변경 전 (약 261번째 줄)
git push https://${GIT_USER}:${GIT_PASSWORD}@github.com/seongtaemin/3tier-app.git HEAD:deploy-${env.TARGET_NAMESPACE} -f

# 변경 후 (실제 Gitea 주소로 변경. 중간에 계정 변수는 그대로 유지)
git push http://${GIT_USER}:${GIT_PASSWORD}@<Gitea서버IP:포트>/<사용자명>/3tier-app.git HEAD:deploy-${env.TARGET_NAMESPACE} -f
